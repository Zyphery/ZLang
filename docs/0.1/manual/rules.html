Types

In ZLang, types play a crucial role in defining the nature of variables and data structures. They bring a sense of clarity and precision to the language, allowing developers to specify the characteristics of their data. ZLang supports a variety of types, including fundamental types like integers, real numbers, and strings. Understanding how to work with these types is essential for effective programming in ZLang.

Variables

Variables are containers for storing data values. In ZLang, each variable must be explicitly assigned a type during its declaration. This ensures that the interpreter can allocate the correct amount of memory and perform operations with precision. For example:

# Declare and initialize an integer variable myVar = int(42) # Declare and initialize a real variable with mutation myReal = real(3.14) : double

Defining a Type

In ZLang, defining a type for variables involves specifying the type and any necessary mutations during their declaration. This is particularly important for achieving precision and clarity in variable assignments. Let's delve into the concept using the real type as an example.

# Declare and initialize a real number a = real(0.78)

Here, a is a variable of type real, representing a floating-point number. The real(0.78) part is a type constructor, and it serves two main purposes:

  • Type Specification: The real keyword indicates that the variable a is of the real number type. In ZLang, various types such as int, string, and custom user-defined types can be used.

  • Initialization: The value 0.78 is the initial value assigned to the variable. This part ensures that a starts with the specified numeric value.

The real(0.78) construct acts as a concise way to declare a variable with a specific type and initial value. It not only communicates the type of data expected but also initializes the variable with the desired value.
For users familiar with other programming languages, this syntax might resemble type casting or conversion. However, in ZLang, it is a combination of type specification and initialization in a single expression.

Explanation:

  • real: Specifies the type of the variable as a real number.

  • (0.78): Initializes the variable with the value 0.78.

The combination of type specification and initialization in a single line simplifies the declaration process. Users can easily communicate the intended type and starting value for their variables, contributing to code clarity and conciseness.

Int

Integers in ZLang are fundamental numeric types used for representing whole numbers. They can be initialized and mutated based on the desired precision:

# Declare and initialize an integer a = int(10) # Declare and initialize an integer with mutation b = int(20) : short

Here, myInt is a regular integer, while myMutatedInt is mutated to have short precision. The ability to mutate types adds flexibility to variable declarations.

Real

Real numbers in ZLang represent floating-point values. Developers can control the precision of real numbers through mutations:

# Declare and initialize a real number myReal = real(3.14) # Declare and initialize a real number with double precision myDouble = real(2.718) : double

The real type supports mutations like double or single, allowing developers to fine-tune the precision based on their requirements.

String

Strings are sequences of characters in ZLang. They are defined using quotes and can be manipulated using built-in string functions:

include usr/std.zlng # Include the standard library for string tools use zstd # Use the scope or namespace zstd (The main scope of the std.zlng library) # Declare and initialize a string myString = "Hello, World!" # Use a string function to get the length stringLength = string.len(myString)

In this example, myString is a string variable, and the string.len function is applied to determine its length. To use some string functions, you must include the standard library for string tools, and use the scope or namespace zstd to access the functions. The zstd namespace is the main scope of the std.zlng library.

Understanding types, variables, and how to define custom types is foundational for ZLang programming. Explicit type declarations enhance code readability and provide a clear structure for data representation. The ability to mutate types adds a layer of flexibility, allowing developers to tailor their data types to specific needs.
